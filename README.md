# Imitation Learning — Applied Project 3

Comparison of three imitation learning algorithms across two continuous-action environments:

| Algorithm | Description |
|-----------|-------------|
| **IQ-Learn** | Inverse soft-Q learning (offline IL) |
| **ML-IRL** | MaxEnt IRL with a standard Q-function (q1) |
| **Soar+ML-IRL** | MaxEnt IRL with a Soar-style Q-function (q4) |

Environments: **CartPoleContinuous-v0** and **Pendulum-v1**, with dataset sizes K ∈ {1, 5, 10, 20, 50} expert trajectories and 3 random seeds each.

---

## Project structure

```
.
├── envs.py                    # ContinuousCartPoleWrapper (Box[-1,1] action space)
├── expert_training.py         # Step 1 & 2: train expert policies
├── dataset_generation.py      # Step 3: roll out experts, save datasets
├── plot_generator.py          # Generate comparison plots from CSV results
│
├── models/                    # Saved SB3 expert models (.zip)
├── expert_data/               # SOAR-IL format tensors
│   ├── states/                #   {env}_K{k}.pt   shape (K, T, obs_dim)
│   └── actions/               #   {env}_K{k}.pt   shape (K, T, act_dim)
├── expert_demos/              # IQ-Learn format: {env}_K{k}.pkl
│
├── algorithm_csv_results/     # Training logs (zipped)
│   ├── cartpole_iq.zip        #   IQ-Learn CartPole results
│   ├── cartpole.zip           #   MaxEntIRL CartPole results
│   ├── pendulum_iq.zip        #   IQ-Learn Pendulum results
│   └── pendulum.zip           #   MaxEntIRL Pendulum results
│
└── report_plots/              # Output figures (generated by plot_generator.py)
```

## Workflow

### Step 1 & 2 — Train expert policies

```bash
python expert_training.py                    # train all environments
python expert_training.py --env cartpole_sac # CartPole with SAC + continuous wrapper
python expert_training.py --env pendulum     # Pendulum with SAC
python expert_training.py --env mcc          # MountainCarContinuous with TD3 + OU noise
```

Saved models go to `models/`. CartPole uses a `ContinuousCartPoleWrapper` so that SAC (required by the IL algorithms) can train on it.

| Environment | Algorithm | Target return |
|-------------|-----------|---------------|
| CartPole-v1 | SAC + continuous wrapper | 500 |
| Pendulum-v1 | SAC | > −200 |
| MountainCarContinuous-v0 | TD3 + OU noise | > 90 |

### Step 3 — Generate expert datasets

```bash
python dataset_generation.py                 # all environments
python dataset_generation.py --env pendulum  # one environment
python dataset_generation.py --env pendulum --inspect  # also print file summary
```

For each environment, K ∈ {1, 5, 10, 20, 50} datasets are saved in two formats:

- **SOAR-IL** (`.pt` tensors): `expert_data/states/` and `expert_data/actions/`
- **IQ-Learn** (`.pkl` dicts): `expert_demos/`

### Step 4 — Train IL algorithms

#### ML-IRL and Soar+ML-IRL (SOAR-IL)

```bash
git clone https://github.com/aidawm/SOAR-IL.git
cd SOAR-IL
pip install -r requirements.txt
```

Copy the SOAR-IL format datasets generated in Step 3:

```bash
cp -r ../expert_data/states  expert_data/states
cp -r ../expert_data/actions expert_data/actions
```

Run the experiments:

```bash
bash run_cartpole_experiments.sh
bash run_pendulum_experiments.sh
```

Results are saved under the repo as `CartPoleContinuous-v0/` and `Pendulum-v1/` with the structure `exp-{K}/maxentirl/{timestamp}_{q_type}_seed{N}_qstd1.0/progress.csv`.

---

#### IQ-Learn

```bash
git clone https://github.com/aidawm/IQ-Learn.git
cd IQ-Learn/iq_learn
pip install -r requirements.txt
```

Copy the IQ-Learn format datasets generated in Step 3:

```bash
cp ../../expert_demos/*.pkl experts/
```

Run the experiments:

```bash
bash scripts/run_cartpole.sh
bash scripts/run_pendulum.sh
```

Then convert the raw logs to `progress.csv` files:

```bash
python generate_progress_csv.py cartpole
python generate_progress_csv.py pendulum
```

Results are saved as `logs/{env}/exp-{K}/iq_learn/seed{N}/progress.csv`.

---

#### Collect results for plotting

Zip each set of results and place them in `algorithm_csv_results/`:

```bash
# from inside SOAR-IL/
zip -r ../algorithm_csv_results/cartpole.zip CartPoleContinuous-v0/
zip -r ../algorithm_csv_results/pendulum.zip Pendulum-v1/

# from inside IQ-Learn/iq_learn/
zip -r ../../algorithm_csv_results/cartpole_iq.zip logs/CartPoleContinuous-v0/ --include "*/progress.csv"
zip -r ../../algorithm_csv_results/pendulum_iq.zip logs/Pendulum-v1/          --include "*/progress.csv"
```

Rename the IQ-Learn log folders to match the expected layout (`cartpole_iq/exp-{K}/...` and `pendulum_iq/exp-{K}/...`) if they differ.

### Step 5 — Generate plots

```bash
python plot_generator.py
```

Reads directly from the zip archives in `algorithm_csv_results/` and writes four figures to `report_plots/`:

| File | Description |
|------|-------------|
| `normalized_performance_vs_k.png` | Normalized score vs K (log-scale x) |
| `mean_return_bar_vs_k.png` | Grouped bar chart of final return per K |
| `learning_curves_all.png` | Training curves for all methods and environments |
| `{env}_mlirl_vs_soar_per_k.png` | ML-IRL vs Soar+ML-IRL per k value |
